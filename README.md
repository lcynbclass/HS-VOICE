# 🌌 HS-VOICE: Honkai Star Rail Voice Analyst
**基于 LLM 的《崩坏：星穹铁道》自动化舆情分析与决策辅助工具**

---

## 🌟 项目简介
在《崩坏：星穹铁道》的长线运营中，社区（B站、米游社、Reddit等）每日产生数以万计的非结构化玩家反馈。传统的“人工抽样+手动打标”模式存在**时效性差、归因主观、难以量化**等痛点。

**HS-VOICE** 旨在利用大语言模型（LLM）的理解能力，实现从“原始评论”到“运营策略建议”的分钟级闭环，辅助项目组进行智能化、数据驱动的舆情决策。

---

## 🚀 核心功能
- **🎯 深度业务归因**: 自动将评论划分为 `[角色与强度, 美术与演出, 福利与抽卡, 玩法与机制, 基础优化]` 等核心运营维度。
- **🎭 情绪健康度监测**: 利用 **Gemini 2.5-flash** 实现精准的情绪打标（正面、负面、中立），实时预警版本口碑危机。
- **💡 策略自动生成**: 针对每一条玩家痛点，AI 自动提炼核心诉求并产出可落地的 **Actionable Insight**（应对策略）。
- **📊 实时看板展示**: 基于 Streamlit 构建的可视化大屏，支持分布统计、负面舆情拦截与数据全量导出。

---

## 🛠️ 技术栈
- **LLM 引擎**: Google Gemini 2.5-flash (2026 最新版本)
- **前端交互**: Streamlit
- **数据处理**: Pandas
- **网络层**: 针对 API Rate Limit 设计的高可用请求调度逻辑（单条请求停顿 5s 以适配免费版 API）。

# 🚀 项目操作指南与开发手册

本手册包含了从本地启动到云端同步的全流程操作建议。

---

## 💻 操作指南 (Usage Guide)

### 第一步：启动应用
在终端执行以下命令开启 Web 界面：

```bash
streamlit run app.py
```

### 第二步：系统配置
在浏览器打开生成的地址后，在侧边栏完成以下操作：

* **API Key**: 输入您的 Google Gemini API Key。
* **代理端口**: 若在中国大陆运行，请输入您的代理端口（如 7890）。

### 第三步：上传并分析
1.  上传本仓库提供的样例数据集 `test_comments.csv`。
2.  点击 “**🚀 启动智能分析**”。
3.  等待分析完成后，即可在下方查看到可视化的舆情大盘及具体的运营建议。

## 📉 运营价值 (Operational Impact)

---

* **降本增效**: 自动化替代 **80%** 的初级舆情打标工作，显著提升监测频率与响应速度。
* **客观归因**: 规避人工抽样产生的“幸存者偏差”，提供覆盖全平台的客观数据视角。
* **闭环决策**: 直接产出应对建议，缩短从“玩家发声”到“官方回应”的内部决策链路。
